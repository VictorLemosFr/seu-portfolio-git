O que eu tiro dessa atividade é que você nunca sabe o quanto você não sabe até você testar. Eu tinha experiência em Git por causa do projeto de IP, mas foi principalmente pelo app de desktop, dessa vez eu tentei fazer da mesma forma e sofri bastante, então tentei aprender a fazer via terminal e aos poucos fui entendendo. Tive dificuldade na hora de editar o Md e decidir qual descrição colocar, eu não sabia se colocava (docs) ou (feat), por muitas vezes acabei colocando feat mesmo ao mesmo tempo que também tive dificuladade de entender o workflow do terminal, quais comandos utilizar primeiro, como voltar um arquivo, quando ir para o site do github e tudo mais e apesar de saber que eu ainda não domino o terminal é bom saber que eu consigo pelo menos me virar agora.

O que foi difícil: Me adaptar à sintax no início e entender o sequência de comandos. Git checkout main -> git add . -> git commit -m "exemplo" -> git push origin main. Foi bem desafiador no começo, mas ao passar do tempo se tornou fácil. Acabou que em um momento eu dei o mesmo commit duas vezes porque eu não sabia que ao criar pastas vazias no github elas não apareciam no site, então acabou que eu usei o comando 2 vezes achando que não tinha funcionado, então também tive que consertar isso depois. A diferença de Docs e Feat foi complicada já que eu não escrevi código propriamente dito, então se eu só usava readme para representar que eu aprendi eu não sabia se era feat ou docs. Também foi difícil pra mim entender que descrição que eu tava escrevendo no site do github. Acabava que eu escrevia uma descrição e percebia depois que tinha sido um comentário, me perdi bastante nisso. Também achei difícil fazer uma descrição "completa" dos meus commits já que eram bem básicos. Acho que a atividade ser mais aberta me confundiu um pouco.

O que ficou claro: Eu entendi de verdade agora o workflow do git, e comentário básicos como: git add .
git pull origin main
git push origin main
git commit -m "efefjeifeji"
git merge
git checkout
etc
Também aprendi a como usar o VS code + terminal + site de forma eficiente, creio que mais eficiente do que quando eu usava o app do Github. Também aprendi as "boas práticas" de commits. feat, fix, docs etc.

O que ainda é confuso: Ainda é confuso para mim a diferença de comentário, descrição, descrição extendida, titulo descritivo, descrição de Pull Request etc. Isso não ficou completamente claro para mim. Outra coisa, eu tinha fechado a atividade eu fui olhar as branches e não ta aparecendo a branch feat/versao-alice, apesar de eu ter certeza que ela foi criada e que eu dei merge nela, mas ao mesmo tempo a branch feat/versao-joao ainda existe, realmente não sei o que aconteceu.
